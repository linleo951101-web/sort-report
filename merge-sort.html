🔀 合併排序法

Merge Sort - 分治策略與穩定排序實作

算法原理

合併排序（Merge Sort）是一種基於分治思想的排序算法。它將原始數列不斷分割成更小的子序列，對子序列進行排序，然後再將已排序的子序列合併成完整的有序序列。

分治思想

  1. 將數列從中間切分為左右兩個子序列
  2. 遞歸地對左右子序列進行合併排序
  3. 將兩個已排序的子序列合併成一個有序序列

穩定性

合併排序是一種穩定排序算法，因為在合併過程中遇到兩個相等元素時，通常會保持它們原本的相對順序。

算法步驟

algorithm MergeSort(array)
    if length(array) <= 1
        return array
    mid = floor(length(array) / 2)
    left = MergeSort(array[0:mid])
    right = MergeSort(array[mid:])
    return Merge(left, right)

algorithm Merge(left, right)
    result = []
    while left not empty and right not empty
        if first(left) <= first(right)
            append first(left) to result
            remove first(left)
        else
            append first(right) to result
            remove first(right)
    append remaining elements from left or right to result
    return result

示例流程

原始序列： [7, 3, 5, 1, 9, 4]

分割過程： [7, 3, 5] 和 [1, 9, 4]

遞歸排序： [3, 5, 7] 和 [1, 4, 9]

合併結果： [1, 3, 4, 5, 7, 9]

代碼實現

C 語言版本

#include <stdio.h>
#include <stdlib.h>

void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;
    int *L = malloc(n1 * sizeof(int));
    int *R = malloc(n2 * sizeof(int));

    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k++] = L[i++];
        } else {
            arr[k++] = R[j++];
        }
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];

    free(L);
    free(R);
}

void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);
    }
}

int main() {
    int data[] = {7, 3, 5, 1, 9, 4};
    int size = sizeof(data) / sizeof(data[0]);
    mergeSort(data, 0, size - 1);
    for (int i = 0; i < size; i++) {
        printf("%d ", data[i]);
    }
    return 0;
}

時間與空間複雜度

項目 最佳情況 平均情況 最壞情況
時間複雜度 O(n log n) O(n log n) O(n log n)
空間複雜度 O(n)（額外暫存空間）
穩定性 穩定

優點

  • 時間複雜度穩定，適合大型數據集
  • 排序穩定，可保持相等元素順序
  • 分治策略概念清晰

缺點

  • 需要額外空間存放暫存陣列
  • 實現較快速排序稍複雜
  • 在小數據集上不一定最有效

應用場景

合併排序適用於需要穩定排序或大量資料處理的情境。典型應用包括：

  • 大規模資料庫排序與外部排序
  • 合併多個已排序清單
  • 分治式演算法底層排序步驟
  • 資料分析與統計計算中的穩定排序需求

總結

合併排序是一種穩定且時間複雜度穩定的排序算法。它特別適合大型資料集與已排序子列的合併操作，但需要額外的暫存空間。

透過分治策略，合併排序將問題拆解為更簡單的子問題，再將結果組合回來，這使得它在處理複雜資料時更具可預測性。
