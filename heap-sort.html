🏗️ 堆排序法

Heap Sort - 以最大堆維持高效率的就地排序演算法

算法原理

基本概念

堆排序（Heap Sort）利用二元堆（Binary Heap）結構進行排序。常見做法是先建立「最大堆」，使根節點始終為目前最大值，接著反覆把根節點與堆尾交換並縮小堆範圍，再重新維持堆性質，直到全部有序。

工作原理

  1. 建堆（Build Heap）：由最後一個非葉節點開始，往前做 heapify，建立最大堆
  2. 取出最大值：將堆頂（最大值）與當前堆尾交換
  3. 縮小堆大小：已放到尾端的元素視為排序完成
  4. 重新堆化：對新根節點做 heapify，恢復最大堆
  5. 重複直到完成：堆大小降到 1 時，整體排序完成

算法步驟

algorithm HeapSort(array)
    buildMaxHeap(array)
    for end from n - 1 downto 1
        swap array[0], array[end]
        heapify(array, 0, end)
    end for
end algorithm

具體例子

對數列 [6, 3, 9, 1, 8, 2, 7] 進行堆排序：

初始數列： [6, 3, 9, 1, 8, 2, 7]
先建立最大堆
建堆後： [9, 8, 7, 1, 3, 2, 6]
堆頂 9 為目前最大值
第一輪提取： 交換堆頂與尾端，得到 [..., 9]
對剩餘區間重新 heapify
重複提取： 每一輪都把當前最大值放到正確尾端位置
最終結果： [1, 2, 3, 6, 7, 8, 9]

代碼實現

C 語言實現

#include <stdio.h>

void heapify(int arr[], int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        int temp = arr[i];
        arr[i] = arr[largest];
        arr[largest] = temp;
        heapify(arr, n, largest);
    }
}

void heapSort(int arr[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    for (int i = n - 1; i > 0; i--) {
        int temp = arr[0];
        arr[0] = arr[i];
        arr[i] = temp;
        heapify(arr, i, 0);
    }
}

C++ 語言實現

#include <iostream>
using namespace std;

void heapify(int arr[], int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, n, largest);
    }
}

void heapSort(int arr[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    for (int i = n - 1; i > 0; i--) {
        swap(arr[0], arr[i]);
        heapify(arr, i, 0);
    }
}

Python 語言實現

def heap_sort(arr):
    def heapify(a, n, i):
        largest = i
        left = 2 * i + 1
        right = 2 * i + 2

        if left < n and a[left] > a[largest]:
            largest = left
        if right < n and a[right] > a[largest]:
            largest = right

        if largest != i:
            a[i], a[largest] = a[largest], a[i]
            heapify(a, n, largest)

    n = len(arr)

    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)

    for end in range(n - 1, 0, -1):
        arr[0], arr[end] = arr[end], arr[0]
        heapify(arr, end, 0)

    return arr

# 使用範例
arr = [6, 3, 9, 1, 8, 2, 7]
print("排序後：", heap_sort(arr))

性能分析

分析項目 時間複雜度 說明
最佳情況 O(n log n) 即使資料接近有序，仍需維持堆操作
平均情況 O(n log n) 建堆 O(n) + n 次提取與堆化
最壞情況 O(n log n) 堆排序可提供穩定的上界表現
空間複雜度 O(1) 原地排序，不需額外陣列
穩定性 不穩定 相等元素相對順序可能改變

時間複雜度分析

  • 建堆成本：自底向上建最大堆，總成本為 O(n)
  • 提取成本：每輪提取後 heapify 約 O(log n)
  • 總成本：n 輪提取，整體為 O(n log n)

性能特點

  • 最壞情況仍維持 O(n log n)，效能上界明確
  • 原地排序，記憶體需求低
  • 不需要遞迴分割整個陣列區間
  • 對快取區域性通常不如快速排序友善

優缺點分析

✓ 優點

  • 最佳、平均、最壞時間皆為 O(n log n)
  • 原地排序，空間複雜度 O(1)
  • 不依賴資料分布，效能可預測
  • 適合需保證最壞時間上限的場景
  • 可作為 introsort 的備援策略

✗ 缺點

  • 不穩定排序，無法保證等值元素順序
  • 常數因子在部分情境較高
  • 資料存取跳躍，快取友善度較差
  • 對小型資料不一定比插入排序快
  • 可讀性通常低於簡單排序法

應用領域

適用場景

  • 最壞情況有硬性要求：需保證上界時可優先考慮
  • 記憶體受限環境：原地排序優勢明顯
  • 混合排序框架：作為退化時的穩健後備策略
  • 系統層任務：適合穩定可預測時間需求

不適用場景

  • 需要穩定排序：可改用歸併排序或 Timsort
  • 極度追求平均常數效能：快速排序常更具優勢
  • 大量重複鍵且需保序：堆排序不適合

總結

堆排序是一個強調「最壞情況保證」與「低空間成本」的經典排序法。當需求重視效能上界與記憶體控制時，堆排序是實務上非常可靠的選擇。若再搭配混合排序策略，能同時兼顧平均效能與穩定上界。

  • 上界穩定：最壞情況仍為 O(n log n)
  • 空間節省：原地排序，額外記憶體極低
  • 工程可靠：常作為高風險退化場景的保險策略
  • 不穩定排序：若需保序需改用其他算法
  • 適合混合應用：可與 quicksort 結合提升整體韌性
