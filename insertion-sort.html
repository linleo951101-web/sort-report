📥 插入排序法

Insertion Sort - 高效的排序算法原理與應用

算法原理

基本概念

插入排序（Insertion Sort）是一種簡單且高效的排序算法，特別適合小型數據集。它的工作方式類似於整理撲克牌：將數列分為已排序和未排序兩部分，每次從未排序部分中取出一個元素，將其插入到已排序部分的適當位置，直到所有元素都被處理完成。

工作原理

  1. 初始化：將第一個元素視為已排序
  2. 選擇元素：從未排序部分的第一個元素開始
  3. 比較和移動：將該元素與已排序部分的元素進行比較，找到適當的位置
  4. 插入元素：將元素插入到找到的位置
  5. 重複過程：對未排序部分的每個元素重複上述操作，直到整個數列排序完成

算法步驟

algorithm InsertionSort(array)
    n = length(array)
    for i = 1 to n-1
        key = array[i]
        j = i - 1
        while j >= 0 and array[j] > key
            array[j+1] = array[j]
            j = j - 1
        end while
        array[j+1] = key
    end for
end algorithm

具體例子

對數列 [5, 2, 8, 1, 9] 進行排序：

初始數列： [5, 2, 8, 1, 9]
已排序部分：[5]，未排序部分：[2, 8, 1, 9]
第一步： 插入 2
比較：2 < 5，2 應插在 5 前
結果：[2, 5, 8, 1, 9]
已排序部分：[2, 5]，未排序部分：[8, 1, 9]
第二步： 插入 8
比較：8 > 5，8 已在正確位置
結果：[2, 5, 8, 1, 9]
已排序部分：[2, 5, 8]，未排序部分：[1, 9]
第三步： 插入 1
比較：1 < 8，1 < 5，1 < 2，1 應插在最前
結果：[1, 2, 5, 8, 9]
已排序部分：[1, 2, 5, 8]，未排序部分：[9]
第四步： 插入 9
比較：9 > 8，9 已在正確位置
最終結果：[1, 2, 5, 8, 9]◎

代碼實現

C 語言實現

#include <stdio.h>

void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        
        // 將大於 key 的元素向後移動
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        
        // 將 key 插入到適當位置
        arr[j + 1] = key;
    }
}

int main() {
    int arr[] = {64, 34, 25, 12, 22, 11, 90};
    int n = sizeof(arr) / sizeof(arr[0]);
    
    insertionSort(arr, n);
    
    printf("排序後的數組：");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
    return 0;
}

性能分析

分析項目 時間複雜度 說明
最佳情況 O(n) 數列已排序，每個元素只需比較一次
平均情況 O(n²) 典型的隨機排列情況
最壞情況 O(n²) 數列逆序排列，每個元素都需要與所有前面的元素比較
空間複雜度 O(1) 只需要固定的額外空間
穩定性 穩定 相等元素的相對順序不變

時間複雜度分析

  • 比較次數：最好情況 n-1 次，最壞情況 1+2+...+(n-1) = n(n-1)/2 次
  • 移動次數：最好情況 0 次，最壞情況 n(n-1)/2 次
  • 總操作數：O(n²) 量級

性能特點

  • 對小規模數據集效率很高
  • 對部分有序的數據特別有效
  • 比氣泡排序更高效（常數因子更小）
  • 在實踐中常用於 O(n log n) 算法的基礎（如快速排序）

優缺點分析

✓ 優點

  • 實現簡單，代碼量少
  • 對小規模數據非常高效
  • 無需額外空間，原地排序
  • 穩定排序算法
  • 自適應性強，對部分有序數據效率高
  • 在線排序（可邊接收邊排序）

✗ 缺點

  • 時間複雜度為 O(n²)
  • 不適合大規模數據排序
  • 移動元素次數較多
  • 性能遠不如快速排序、歸併排序
  • 不適合實時性要求高的應用

應用領域

適用場景

  • 小規模數據：數據量 < 1000 時效率很好
  • 部分有序數據：自適應性強，效率相對較高
  • 在線排序：接收數據即可排序，不需提前獲得所有數據
  • 內存限制：只需 O(1) 的額外空間
  • 穩定排序需求：需要保持相等元素的相對順序
  • 教學用途：算法簡單易懂，是學習排序的好材料

不適用場景

  • 大規模數據排序：O(n²) 時間複雜度不可接受
  • 實時系統：無法滿足嚴格的性能要求
  • 競賽編程：通常會因時間限制而超時
  • 生產環境：應優先使用已優化的標準庫函數

實際應用示例

// 示例：對學生成績單進行排序
typedef struct {
    char name[50];
    int score;
    char grade;
} Student;

void sortStudentsByScore(Student students[], int n) {
    for (int i = 1; i < n; i++) {
        Student key = students[i];
        int j = i - 1;
        
        while (j >= 0 && students[j].score > key.score) {
            students[j + 1] = students[j];
            j--;
        }
        
        students[j + 1] = key;
    }
}

總結

插入排序是一種兼具簡單性和實用性的排序算法，具有以下特點：

  • 實現簡單：代碼簡潔易懂，容易理解算法原理
  • 自適應性強：對部分有序的數據效率特別高
  • 空間高效：只需 O(1) 的額外空間
  • 穩定性好：能夠保持相等元素的相對順序
  • 在線排序：可以邊接收數據邊進行排序
  • 時間效率低：O(n²) 的時間複雜度限制其應用範圍
  • 實用價值高：常作為大規模算法的基礎或優化手段

建議：在實際應用中，對小規模數據（< 1000 個元素）使用插入排序是個不錯的選擇。對於大規模數據，應優先使用快速排序、歸併排序或使用標準庫提供的優化排序函數。許多高效排序算法（如 Timsort）內部就使用了插入排序作為優化策略。
