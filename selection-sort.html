Selection Sort - 基礎排序算法的原理與應用

算法原理

基本概念

選擇排序（Selection Sort）是一種簡單直觀的排序算法，它通過不斷尋找未排序部分中的最小元素，並將其放在已排序部分的末尾，直到整個數列排序完成。算法名稱來源於其不斷"選擇"最小元素的工作特點。

工作原理

  1. 初始化：第一個位置是已排序部分，其餘都是未排序部分
  2. 尋找最小值：在未排序部分中找到最小元素
  3. 交換位置：將最小元素與未排序部分的第一個元素交換
  4. 擴展已排序區域：已排序部分擴展一個元素
  5. 重複過程：重複上述步驟，直到整個數列排序完成

算法步驟

algorithm SelectionSort(array)
    n = length(array)
    for i = 0 to n-1
        minIndex = i
        for j = i+1 to n-1
            if array[j] < array[minIndex]
                minIndex = j
            end if
        end for
        swap(array[i], array[minIndex])
    end for
end algorithm

具體例子

對數列 [5, 2, 8, 1, 9] 進行排序：

初始數列： [5, 2, 8, 1, 9]
第一輪：
尋找最小值：1 (位置 3)
交換 array[0] 和 array[3] → [1, 2, 8, 5, 9]
已排序部分：[1]
第二輪：
尋找最小值：2 (位置 1)
交換 array[1] 和 array[1] → [1, 2, 8, 5, 9]
已排序部分：[1, 2]
第三輪：
尋找最小值：5 (位置 3)
交換 array[2] 和 array[3] → [1, 2, 5, 8, 9]
已排序部分：[1, 2, 5]
第四輪：
尋找最小值：8 (位置 3)
交換 array[3] 和 array[3] → [1, 2, 5, 8, 9]
已排序部分：[1, 2, 5, 8]
最終結果： [1, 2, 5, 8, 9]

代碼實現

C 語言實現

#include <stdio.h>

void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        // 尋找最小元素
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        // 交換位置
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}

int main() {
    int arr[] = {64, 34, 25, 12, 22, 11, 90};
    int n = sizeof(arr) / sizeof(arr[0]);
    
    selectionSort(arr, n);
    
    printf("排序後的數組：");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
    return 0;
}

Python 語言實現

def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_index = i
        # 尋找最小元素
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j
        # 交換位置
        arr[i], arr[min_index] = arr[min_index], arr[i]
    return arr

# 使用示例
arr = [64, 34, 25, 12, 22, 11, 90]
print("排序後的數組：", selection_sort(arr))

public class SelectionSort {
    public static void selectionSort(int[] arr) {
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            int minIndex = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
            // 交換
            int temp = arr[i];
            arr[i] = arr[minIndex];
            arr[minIndex] = temp;
        }
    }

        }
    }

性能分析

分析項目 時間複雜度 說明
最佳情況 O(n²) 即使數列已排序，仍需進行 n(n-1)/2 次比較
平均情況 O(n²) 隨機排列情況下的典型性能
最壞情況 O(n²) 數列逆序排列，需要最多比較
空間複雜度 O(1) 只需要固定的額外空間
穩定性 不穩定 交換操作可能改變相等元素的相對順序

時間複雜度分析

  • 比較次數：無論任何情況下都是 (n-1) + (n-2) + ... + 1 = n(n-1)/2 次
  • 交換次數：最多 n-1 次（因為每輪循環最多交換一次）
  • 總操作數：O(n²) 量級，且不會因為數據已部分排序而改善

性能特點

  • 選擇排序的時間複雜度 恆為 O(n²)，無最佳情況
  • 關鍵操作次數少：交換次數最多為 n-1 次，少於氣泡排序
  • 比較次數固定，約為 n²/2 次，這決定了其性能上限

優缺點分析

✓ 優點

  • 實現簡單，容易理解
  • 無需額外空間，原地排序
  • 交換次數少（最多 n-1 次）
  • 對小規模數據效率可接受
  • 選擇是最小化交換操作的算法

✗ 缺點

  • 時間複雜度恆為 O(n²)
  • 無法藉助已排序數據進行優化
  • 大規模數據排序效率低下
  • 比較次數多（n²/2 次）
  • 排序不穩定

應用領域

適用場景

  • 教育目的：作為排序算法的基礎學習材料
  • 小規模數據：數據量較小（通常 < 1000）時可使用
  • 交換操作昂貴：當交換操作成本很高時，選擇排序由於交換次數少而較優
  • 內存限制嚴格：只需 O(1) 的額外空間
  • 部分排序任務：尋找前 K 個最小元素

不適用場景

  • 大規模數據排序：性能瓶頸明顯
  • 實時系統：不能滿足性能要求
  • 需要穩定排序：選擇排序是不穩定的
  • 競賽編程：時間限制會導致超時
  • 生產環境：應使用更高效的算法

實際應用示例

// 實際應用：尋找 Top-K 最小元素
#include <stdio.h>
#include <string.h>

typedef struct {
    char name[50];
    int score;
} Student;

// 選擇排序：找出成績前 3 低的學生
void findLowestScores(Student students[], int n, int k) {
    for (int i = 0; i < k && i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (students[j].score < students[minIndex].score) {
                minIndex = j;
            }
        }
        Student temp = students[i];
        students[i] = students[minIndex];
        students[minIndex] = temp;
    }
}

int main() {
    Student students[] = {
        {"Alice", 85},
        {"Bob", 92},
        {"Charlie", 78},
        {"David", 88},
        {"Eve", 75}
    };
    int n = 5;
    
    findLowestScores(students, n, 3);
    
    printf("成績最低的 3 名學生：\n");
    for (int i = 0; i < 3; i++) {
        printf("%s: %d\n", students[i].name, students[i].score);
    }
    return 0;
}

與其他排序算法的比較

算法名稱 最佳情況 平均情況 最壞情況 空間 穩定性 交換次數
氣泡排序 O(n) O(n²) O(n²) O(1) 穩定 O(n²)
選擇排序 O(n²) O(n²) O(n²) O(1) 不穩定 O(n)
插入排序 O(n) O(n²) O(n²) O(1) 穩定 O(n²)
快速排序 O(n log n) O(n log n) O(n²) O(log n) 不穩定 O(n log n)
歸併排序 O(n log n) O(n log n) O(n log n) O(n) 穩定 O(n log n)
堆排序 O(n log n) O(n log n) O(n log n) O(1) 不穩定 O(n log n)

對比分析

  • vs 氣泡排序：選擇排序交換次數更少，但不穩定；氣泡排序在已排序數據上性能更好
  • vs 插入排序：插入排序在部分排序數據上性能更優；選擇排序交換次數更少
  • vs 高效算法：快速排序、歸併排序是現實應用中的首選，時間複雜度更優

總結

選擇排序是最簡單的排序算法之一，具有以下特點：

  • 算法簡單：代碼容易實現和理解，是學習排序算法的好起點
  • 空間高效：只需 O(1) 的額外空間，適合內存有限的場景
  • 交換最少：交換次數最多 n-1 次，在交換操作昂貴時有優勢
  • 時間複雜度固定：恆為 O(n²)，無法通過優化來改善
  • 排序不穩定：相等元素的相對順序可能改變
  • 實用性有限：在實際應用中應優先選擇快速排序、歸併排序等高效算法

建議：選擇排序最適合作為教學工具和小規模數據排序。在實際開發中，應優先使用標準庫提供的排序函數或選擇時間複雜度為 O(n log n) 的排序算法。
