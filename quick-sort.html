⚡ 快速排序法

Quick Sort - 分治策略的高效排序演算法

算法原理

快速排序（Quick Sort）是一種基於分治法的排序演算法。它選取一個樞紐（pivot），將陣列分成比 pivot 小與大的兩部分，分別遞迴排序，再合併結果。

基本步驟

  1. 選取 pivot（如最後一個元素）
  2. 進行 partition，將小於 pivot 的元素放左邊，大於 pivot 的放右邊
  3. 遞迴對左右子陣列進行快速排序
function quickSort(arr, low, high)
    if low < high
        p = partition(arr, low, high)
        quickSort(arr, low, p - 1)
        quickSort(arr, p + 1, high)
    end if
end function

代碼實現（範例）

Python

def quicksort(arr):
    def partition(a, low, high):
        pivot = a[high]
        i = low - 1
        for j in range(low, high):
            if a[j] <= pivot:
                i += 1
                a[i], a[j] = a[j], a[i]
        a[i+1], a[high] = a[high], a[i+1]
        return i+1

    def qs(a, low, high):
        if low < high:
            p = partition(a, low, high)
            qs(a, low, p-1)
            qs(a, p+1, high)

    qs(arr, 0, len(arr)-1)
    return arr

性能分析

分析項目 時間複雜度 說明
最佳情況 O(n log n) 每次選擇的 pivot 將數列均分
平均情況 O(n log n) 隨機排列的數列
最壞情況 O(n²) 數列已排序或逆序，pivot 總是最小或最大值
空間複雜度 O(log n) 遞迴調用棧的深度
穩定性 不穩定 相等元素的相對順序可能改變

優缺點分析

✓ 優點

  • 平均時間複雜度低，適合大規模數據
  • 原地排序，空間效率高
  • 遞迴實現簡單，易於理解

✗ 缺點

  • 最壞情況時間複雜度高
  • 不穩定，可能改變相等元素的順序
  • 對小規模數據效率不如插入排序

應用領域

適用場景

  • 大規模數據：快速排序在數據量較大時表現優異
  • 隨機排列數據：對隨機數據的平均性能穩定
  • 內存有限：原地排序節省空間

不適用場景

  • 小規模數據：插入排序可能更高效
  • 實時系統：最壞情況可能導致性能瓶頸

優化與應用

分類 內容
原理 快速排序法基於分治法，選取樞紐（pivot）將資料分為兩部分，遞迴排序後合併。
優點 平均時間複雜度為 O(n log n)，適合大部分情況；原地排序，空間效率高。
缺點 最壞情況時間複雜度為 O(n²)，當資料接近有序時效率低。
應用 適用於需要高效排序的大型資料集，如數據分析、資料庫排序。

總結

快速排序在平均情況下提供極佳性能且為原地排序，適合大多數一般用途；開發時宜採用隨機化或混合策略（如 introsort）以避免最壞情況，並在需要穩定性的場景選擇其他演算法。
